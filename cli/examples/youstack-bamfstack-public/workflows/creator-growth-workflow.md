# Creator Growth Workflow

1. Verify the stack with `youmd stack smoke`.
2. Load the capability map before choosing a local skill, REST endpoint, or MCP tool.
3. Inspect creator context: profile, scheduled posts, ideas, knowledge, analytics, media, recent wins, and duplicate risks.
4. Run research when the request depends on current facts, trends, benchmarks, regulations, platform changes, or exact numbers.
5. Run a mini-interview when the draft needs the creator's real story, opinion, mistake, customer moment, or proof.
6. Propose three concepts and any media/carousel plan.
7. Wait for approval of exact content and action.
8. Create or mutate content only through authenticated API/MCP paths and only after approval.
9. Record useful lessons as improvement proposals for the stack, not automatic brain writes.

Quality gate:

- Strong hook.
- Clear stakes line.
- Creator POV.
- Specific proof.
- Source-backed numbers when numbers are used.
- Scannable structure.
- Useful payoff.
- Non-generic close.
