Synthesis of four dimension audits (core product, YouStacks layer, API/MCP surface, CLI/agent integration). All findings spot-checked against the repo at commit `d088ef0`. Overlapping findings across dimensions are deduped; each row cites verifying evidence.

## Strengths

- **Honest, machine-generated surface counts** with sourceHash drift check in CI (`scripts/generate-docs-reference.mjs` → `src/generated/docs-reference.ts`, enforced by `npm run docs:check`); independent recount confirms 76 unique endpoints (59 Convex-served + 26 Next.js-served, 9 shared), 24 CLI MCP tools.
- **Content-addressed version control with real conflict semantics**: ANCESTOR_MISMATCH → 409 (`convex/me.ts:486-492`, `convex/http.ts:521-529`) surfaced in the CLI with pull-first UX, empty-bundle guard, and >50% size-regression guard (`cli/src/commands/push.ts:37-162`).
- **Defense-in-depth on paid endpoints**: per-IP sliding-window rate limits, payload caps, daily spend-cap kill switch with documented threat reasoning (`convex/http.ts:782-857, 1336-1561`, `convex/schema.ts:592-616`, `convex/crons.ts:28-33`).
- **Systematic owner-auth sweep** with candid exploit-cycle history in comments (`convex/memories.ts:9`, `convex/pipeline/index.ts:13-16`, `convex/lib/auth.ts`).
- **Agent-grade HTTP ergonomics where shipped**: contentHash ETag + 304, content negotiation (markdown vs you-md/v1 JSON), JSON Schema Link headers, scoped TTL/max-use context links (`convex/http.ts:58-219`, `convex/contextLinks.ts:23-103`).
- **Cross-agent activity log** with before/after bundle hashes (`convex/schema.ts:557-586`, `convex/http.ts:475-489`) — the right audit substrate, pending attribution fix (#35).
- **Readiness envelopes** in the CLI MCP (auth_required / unavailable / ready) so agents degrade honestly instead of hallucinating (`cli/src/mcp/server.ts:171-369`); same contract published at `GET /api/v1/stacks/capabilities` (`src/lib/youstack-routing.ts:101-121`).
- **Real SSE streaming** with tool_use block transformation for OpenRouter and Anthropic formats (`convex/http.ts:935-1012`).
- **Inbound GitHub webhook with HMAC-SHA256 verification** auto-pulling linked repos into the server mirror (`convex/http.ts:3342-3447`), private/ paths excluded from the mirror (`convex/githubRepo.ts:107-115`).
- **Additive repo bootstrap**: managed `<!-- youmd:bootstrap -->` blocks upsert into existing AGENTS.md/CLAUDE.md, STACK-MAP.md declares ownership, zero-touch mode (`cli/src/lib/skills.ts:493-547, 705-764, 861-881`).
- **Unusually complete agent docs surface**: llms.txt, llms-full.txt, docs reference, OpenAPI, .well-known/mcp.json, capability contract — most products ship none.
- **Decent CLI test breadth** (12 test files: config, compiler, decompile, hash, first-run, MCP brief, integration) and polite update checking (12h cache, 1.5s abort, silent failure).

## Gap Themes

1. **One brain, four bodies.** The same user is a different "you" depending on entry point (web/CLI/hosted MCP/repo). This is the direct negation of the core promise and the single most important product gap (#3, #7, #19).
2. **Contracts that lie.** Scopes recorded but unenforced (#1), draft/rollback semantics that don't do what they say (#2), capability contract pointing at phantom endpoints (#12), MCP install that installs nothing (#8), docs counts conflating local and hosted surfaces (#7, #27). An identity *protocol* lives or dies on contract trustworthiness.
3. **The memory brain can't remember.** No search (#6), category fragmentation (#17), silent decay of durable preferences (#16) — "stop re-explaining yourself" is not yet true past 90 days or 100 memories.
4. **YouStacks are a pitch, not a product.** No install flow, no registry, three layouts, zero identity in adapters, divergent routers (#10, #11, #12, #20, #21).
5. **Data-loss and security footguns in the daily path.** `youmd sync` destroys edits (#5), Math.random tokens (#4), world-readable keys (#30), private context staged for git commit (#37).
6. **Version skew across the runtime itself.** npm 0.6.21 vs repo 0.6.23 vs unpinned source HEAD means agents literally run different versions of the product that delivers "the same version of you" (#9, #36).
