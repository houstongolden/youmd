# 2026-06-17 - Project Portfolio Graph, Dependency Routing, and Reuse Catalog

Source: Houston long-form prompt in the You.md Codex session on 2026-06-17.

## Sanitized Capture

Houston connected the BAMF site/BAMF.ai Lempod duplication problem to a broader
You.md product need: You.md should become the central source of truth for active
projects, APIs, MCPs, skill stacks, protected product agent harnesses, local and
global agent skills, machines, project environments, reusable code patterns, and
reusable UI/UX decisions.

The requested product direction:

- Explain and improve how You.md saves and organizes Projects in a systematic,
  strategic, goal-oriented way for a solo founder using many agents across many
  products.
- Ensure Projects carry both detailed and summarized AI descriptions plus
  maintained PRD, tasks, design, research, ideas, changelog, current state, and
  agent context files.
- Add a dashboard view for APIs/MCPs and associated projects, including primary
  owner project, connected projects, dependency direction, integration tier,
  integration type, notes, features powered, and failure risk.
- Track whether an integration is user-level, admin/developer-agent-only,
  workspace-level, product-internal, or public.
- Preserve the distinction between protected in-product agentic harnesses and
  public/installable skill stacks that teach external agents how to use a
  product API/MCP.
- Use You.md to prevent duplicate APIs/endpoints/skills/stacks across projects,
  with Lempod management across `bamfsite` and `bamfaiapp` as the concrete first
  example to audit.
- Build a cross-project code and UI/UX reuse catalog so new projects can reuse
  strong patterns without reinventing auth, role hierarchy, sidebar layout,
  agentic chat UX, streaming response patterns, connector surfaces, docs, API,
  MCP, stack install, and env practices.
- Preserve Houston's standard left-sidebar/app-shell and agentic split workspace
  preferences as reusable cross-project templates.
- Preserve role hierarchy preferences: solo user, workspace/team/org member,
  manager, admin, owner, and super-admin reserved for Houston/developer agents.
- Preserve auth preferences: custom passwordless email/SMS/iMessage first,
  Convex Auth when multiple SSO providers justify it, custom GitHub OAuth when
  enough, and no Clerk/paid auth provider by default.
- Add a YouStack/skill that audits local projects, prompts, docs, APIs, MCPs,
  and code patterns, then proposes a portfolio graph and reusable pattern plan.

No secrets, credentials, phone numbers, or API keys were included in the prompt.
