### Context-load recipe (agent-facing)
Upstream doc spells out a consistent, low-variance flow:
1. Extract 2–4 keywords (nouns/error names/paths/technical terms).
2. Run brain search.
3. If few hits, broaden search once.
4. Read top 3 results and cite slugs if they mattered.
5. If brain search fails, proceed (no retries inline).

Upstream reference:
- `.reference-repos/garrytan/gstack/docs/gbrain-write-surfaces.md` (§Context Load)

You.md mapping (where to implement later):
- Web “You Agent” planning mode and CLI skill runners should share one helper: keyword extraction → search → fetch top N → attach as context with lightweight provenance.
